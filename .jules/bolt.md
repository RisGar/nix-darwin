## 2024-05-24 - Avoid `cat` subprocesses in shell scripts for reading single files
**Learning:** In Fish and Bash, invoking `cat` inside a command substitution to read a file (Fish: `(cat file)`; Bash: `$(cat file)`) is a common anti-pattern that spawns an unnecessary subprocess. This significantly degrades performance, especially in hot paths like shell initialization scripts where multiple small files (e.g. secrets) are read. I ran a benchmark and found `cat` substitution to be 118x slower than using the builtin redirection (`< file`).
**Action:** When a file's content is needed in a shell script, always use input redirection (e.g., `set val (string trim < file)`) or builtin tools instead of spawning a `cat` subprocess.
